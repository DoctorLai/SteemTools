# Support

Thanks for using **Steem Tools**! This page explains how to get help and where to
report problems.

## Getting help

- **Bugs** — search the [issue tracker](https://github.com/DoctorLai/SteemTools/issues)
  first, then open a new issue using the
  [bug report template](.github/ISSUE_TEMPLATE/bug_report.md).
- **Feature ideas** — open a
  [feature request](.github/ISSUE_TEMPLATE/feature_request.md).
- **Security vulnerabilities** — please **do not** open a public issue. Follow the
  private process in [SECURITY.md](SECURITY.md).
- **More tools & articles** — visit the author's site at
  [steemyy.com](https://steemyy.com).

## Before you open an issue

To help us resolve things quickly, please include:

- The extension version (shown in the popup footer, e.g. `Version: v1.1.0 …`).
- Your browser and version (Chrome or Edge).
- What you did, what you expected, and what actually happened.
- Any errors from the browser console (`F12` → **Console**) and a screenshot if useful.

And make sure you are on the latest version:

1. Update from the Chrome Web Store, or pull the latest `master`.
2. Reload the extension at `chrome://extensions` → **Reload**.

## Response times

Steem Tools is maintained by volunteers in their spare time, so there is no
guaranteed response time. Clear, reproducible reports get resolved fastest.
